auditTransaction

<flow name="auditTransaction">
    <composite-source>
        <vm:inbound-endpoint exchange-pattern="one-way" path="Request"> </vm:inbound-endpoint>
        <vm:inbound-endpoint exchange-pattern="one-way" path="Response">    </vm:inbound-endpoint>
    </composite-source>
    <mulexml:xslt-transformer maxIdleTransformers="2" maxActiveTransformers="5" xsl-file="payload-to-transaction.xslt"> </mulexml:xslt-transformer>
    <flow-ref name="invokeAuditService"/>
</flow>

Picks up request and response messages and invokes the AuditService with them. We can do this generically because the Transaction element in the model schema is extensible with in the message namespace. We determine the type of transaction based on the incoming path on the VM queue.
